\section{Цель работы}

Изучить базовые задачи комбинаторной оптимизации и особенности их решения с помощью генетических алгоритмов. Изучить метод решения задачи коммивояжера с помощью ГА. Изучить основные способы представления особей-туров и операторы рекомбинации для каждого представления. Реализовать ГА, решающий задачу коммивояжера с циклическим кроссинговером (CX).

\section{Теоретические сведения}

\subsection{Задача коммивояжера}

Коммивояжер должен посетить каждый город на своём участке один раз и вернуться в исходную точку. Дано множество городов с координатами $(x_i, y_i)$ на плоскости. Стоимость перехода между городами определяется евклидовым расстоянием:
\begin{equation}
d_{ij} = \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2}.
\end{equation}
Необходимо найти тур (перестановку городов) минимальной суммарной стоимости.

\subsection{Представление путей}

Тур представляется упорядоченным списком городов, входящих в тур. Например, $(5, 1, 7, 8, 9, 4, 6, 2, 3)$ означает маршрут $5 \to 1 \to 7 \to 8 \to 9 \to 4 \to 6 \to 2 \to 3 \to 5$.

\subsection{Циклический кроссинговер (CX)}

Циклический оператор кроссинговера строит потомков так, что каждый город вместе со своей позицией наследуется от одного из родителей:
\begin{enumerate}
\item Начать с первого города родителя~$P_1$ в позиции~1.
\item Найти город родителя~$P_2$ в той же позиции.
\item Найти позицию этого города в~$P_1$ и взять его оттуда.
\item Повторять, пока не образуется цикл.
\item Оставшиеся позиции заполнить из~$P_2$.
\end{enumerate}
Оператор CX сохраняет абсолютные позиции городов из родителей.

\subsection{Мутация обменом (swap)}

Случайно выбираются две позиции в туре, и города в этих позициях обмениваются местами. Такая мутация выполняет локальное изменение тура, сохраняя его допустимость (каждый город посещается ровно один раз).

\section{Ход работы}

\subsection{Тестовые данные}

Использован набор eil76.tsp (76 городов) из библиотеки TSPLIB (Christofides/Eilon). Известный оптимальный тур имеет длину~538.

\subsection{Реализация генетического алгоритма}

Реализован ГА с представлением путей:
\begin{itemize}
\item Инициализация: случайные перестановки $n$ городов.
\item Селекция: турнирная с размером турнира $k = 3$.
\item Кроссинговер: циклический (CX) с вероятностью $P_{cx} = 0.9$.
\item Мутация: обмен пары городов (swap) с вероятностью $P_{mut} = 0.2$.
\item Сокращение: элитарная стратегия.
\end{itemize}

Параметры: размер популяции 150, число поколений 500.

\subsection{Визуализация}

Построены:
\begin{itemize}
\item Карта расположения всех 76 городов.
\item График лучшего и среднего тура по поколениям с отметкой оптимума.
\item Визуализация лучшего найденного тура на карте городов.
\item Анимация улучшения тура по поколениям.
\end{itemize}

\subsection{Анализ гиперпараметров}

Исследовано влияние параметров: размер популяции, вероятность кроссинговера, вероятность мутации, размер турнира. Показано, что высокая вероятность мутации ($P_{mut} = 0.2$) важна для комбинаторных задач, поскольку обеспечивает выход из локальных оптимумов.

\section{Результаты}

\begin{enumerate}
\item ГА с циклическим кроссинговером находит туры, значительно лучше случайных, приближаясь к известному оптимуму.
\item CX-кроссинговер сохраняет абсолютные позиции городов, что делает его эффективным для задач, где важна позиция элемента.
\item Swap-мутация обеспечивает локальное улучшение без разрушения общей структуры тура.
\item Элитарная стратегия гарантирует монотонное убывание длины лучшего тура.
\item Основная сходимость достигается к 200--300 поколениям, дальнейшие улучшения незначительны.
\end{enumerate}

\section{Выводы}

В ходе лабораторной работы реализован генетический алгоритм для решения задачи коммивояжера на тестовом наборе из 76 городов. Изучены особенности представления решений-перестановок и специализированные генетические операторы для комбинаторной оптимизации. Циклический кроссинговер (CX) эффективно передаёт позиционную информацию от родителей к потомкам. Экспериментально показано, что ГА способен находить качественные субоптимальные решения задачи коммивояжера, хотя достижение точного оптимума требует либо бо\textasciiacute{}льших популяций, либо комбинирования с локальным поиском (2-opt, 3-opt).
